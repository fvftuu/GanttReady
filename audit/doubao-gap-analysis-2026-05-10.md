# NetPlan 对标行业需求 — 缺失功能分析

> 来源：[豆包对话](https://www.doubao.com/thread/wf454d9b3c6e73949) — 一位20年工程老兵的NetPlan项目深度辩论  
> 分析日期：2026-05-10

---

## 对话核心结论

经过多轮激烈辩论，AI 最终承认 NetPlan 在核心技术（自动虚工作、双图联动、自动排版、前锋线）上已超越梦龙/Project/斑马。但指出了 **7个致命缺口**，这些才是工程人"愿意付钱"的真正理由。

---

## NetPlan 已具备的优势（辩论中 AI 最终认可）

| 功能 | 状态 | 对话评价 |
|------|:---:|---------|
| 自动生成虚工作 | ✅ | "吊打梦龙和广联达" |
| 甘特图↔网络图实时联动 | ✅ | "降维打击" |
| 自动排版不交叉 | ✅ | "独家技术优势" |
| 前锋线自动计算（前端线+节点着色） | ✅ 但有缺陷 |
| 四种依赖100%识别 (FS/SS/FF/SF) | ✅ |
| 一键调整后续任务+自动重算CP | ✅ |
| 导出HTML交互式查看 | ✅ |
| 20年工程经验注入算法 | ✅ | "唯一不可复制的优势" |

---

## 7 个致命缺口（对话中反复强调）

### 1. 工序库 / 行业模板 → **最大的缺失**

> "工程人要的不是'能输任务'，是点一下，500道工序自动出来"

| 当前状态 | 需要做的 |
|---------|---------|
| 能建任务、输入名称工期 | 按行业（建筑/市政/公路/水利/港航）内置标准工序库 |
| 无自动推荐 | 紧前紧后自动推荐 |
| 无标准工期 | 标准工期自动填充 |
| 无项目模板 | "30层住宅"模板一键生成500+工序+依赖 |

**优先级：P0** — 直接影响新用户留存率

---

### 2. 资源冲突 → 只检测不决策

> "工程人要的不是'这里冲突了'，而是'优先排哪个、挪几天、加几个人'"

| 当前状态 | 需要做的 |
|---------|---------|
| 资源冲突检测 ✅ | 给出具体解决方案（优先排/可推/加人/成本） |
| 资源负载图 ✅ | 一键资源平衡优化 |
| 无指导 | 挪几天不影响总工期？加几个人能解决？ |

**优先级：P1**

---

### 3. 导出/打印模板 → 一套格式不够

> "不要同一个图导出3次改3次格式"

| 当前状态 | 需要做的 |
|---------|---------|
| PNG/PDF 导出 ✅ | 甲方版/监理版/内部版三套显示规则一键切换 |
| 打印预览 ✅ | 不是换皮肤，是换内容（显示哪些字段） |
| 效果固定 | 甲方版：只显示里程碑+关键线路 |
| | 监理版：显示代号+时差+前锋线 |
| | 内部版：显示资源+责任人 |

**优先级：P1**

---

### 4. 工程级打印 → 不是"能打印"，是"直接上交"

> "工程人要的不是屏幕好看，是打印出来直接盖章上交"

| 当前状态 | 需要做的 |
|---------|---------|
| 基础打印 ✅ | 自动分页打印（图签+页码） |
| 缩放调整 ✅ | 智能比例：一键"打印在1张A3上" |
| | 黑白打印优化：不同线型区分任务状态 |
| | 所见即所得的打印预览 |
| | 批量打印所有里程碑节点 |

**优先级：P1**

---

### 5. 批量调整+专业筛选 → 数据量大时的效率

> "甲方说'把土方工程往后推10天'，你能一键筛选专业只调整对应任务吗？"

| 当前状态 | 需要做的 |
|---------|---------|
| 单个任务调整 ✅ | 按专业/类别批量筛选 |
| 一键调整后续 ✅ | 批量修改工期/开始时间/资源 |
| | 春节/雨季场景：一键顺延/调整室外工序 |

**优先级：P2**

---

### 6. 操作习惯兼容 → 老工程师上手

> "梦龙怎么操作你就怎么操作，老工程师不用学拿来就用"

| 当前状态 | 需要做的 |
|---------|---------|
| 自定义UI ✅ | 全键盘快捷键 |
| 右键菜单 ✅ | 右键菜单对齐行业习惯 |
| | 首次打开引导/提示"这个按钮是做什么的" |

**优先级：P2**

---

### 7. 数据积累/闭环 → 核心护城河

> "把上一个项目的实际工期自动同步到模板里，生成更准确的下一个项目计划"

| 当前状态 | 需要做的 |
|---------|---------|
| 单项目使用 | 历史项目数据分析：实际工期 vs 计划工期 |
| | 经验工期库自动更新 |
| | 项目模板保存和复用 |

**优先级：P3（战略级，短期不做）**

## 关键争议和 NetPlan 的回应

| AI 质疑 | NetPlan 回应 |
|---------|-------------|
| 不支持 .mpp/梦龙导入 | XML 格式不分版本，Excel 批量导入即可，开源方案够用 |
| 无行业模板 | 花3天可做，不是壁垒 |
| 质监站要网络图 | 这是 AI 的知识漏洞，质监站不索要网络图 |
| 开源会被大厂抄 | 是最大的真实风险，但在辩论中已被反驳 |

## 建议优先级（从对话中提取）

### 立即（本周）
1. **工序库** — 哪怕只做"建筑-住宅"一个模板，效果立竿见影
2. **导出套版** — 甲方版/内部版两套规则切换

### 短期（1-2周）
3. 资源冲突 → 解决建议
4. 工程打印：分页+图签+页码
5. 批量操作：按专业筛选+批量调整

### 中期
6. 操作习惯兼容（快捷键+引导）
7. 前锋线改造为真折线

## 对话中最致命的点评

> "你做了一个全世界最好的进度工具，但你没有告诉用户'它为什么好'。"
> 
> "你有最好的技术，但没有把它变成用户一眼就能看到的尖叫点。"

这意味着：**产品已 90% 完成，但"首次体验"和"可发现性"不足。**

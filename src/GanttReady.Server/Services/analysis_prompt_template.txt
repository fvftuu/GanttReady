【AI 进度分析报告 — 自定义规则模板】
========================================
使用方法：
1. 用记事本打开本文件
2. 修改下面各章节的内容要求
3. 在系统的 AI 设置中上传此文件
4. 点"AI 分析"即可按你的规则生成报告

注意事项：
- 每行开头的 # 和 【】是注释说明，可以保留或删除
- 「输出要求」和「语言要求」建议保留
- 每个 <h2> 是一大章，<h3> 是小节
- <ul><li> 是列表，<p> 是段落
- 表格用 <table><tr><th><td> 标签
- 🟢🟡🔴 是状态标记

========================================
# ==== 以下是 AI 实际执行的规则 ====

## 输出要求
输出 HTML，不要用 markdown。允许的标签：
- <h2> <h3> 做章节标题
- <table> <tr> <th> <td> 做表格
- <ul><li> 做列表
- <p> <div> 做段落
- <span style="color:green">🟢</span> / <span class="text-warning">🟡</span> / <span style="color:red">🔴</span> 标注状态

# ==== 下面是报告大纲，你可以随意增删改 ====

# 【第一章 项目总览】— 整体概况
<h2>一、项目总览</h2>

# 【1.1 核心结论】一句话说明项目整体状态
<h3>1.1 核心结论</h3>
<p>一句话说明项目整体状态，总进度偏差，是否可控。</p>

# 【1.2 核心指标】用列表展示关键数字
<h3>1.2 核心指标</h3>
<ul>
<li>整体完成度</li>
<li>关键任务完成率</li>
<li>里程碑达成率</li>
<li>总进度偏差</li>
</ul>

# 【1.3 任务状态分类】已完成/进行中/未开始/滞后的数量和占比
<h3>1.3 任务状态分类</h3>
<p>已完成/进行中/未开始/滞后任务的数量和占比。</p>

# 【第二章 详细进度对比】— 关键任务和里程碑
<h2>二、详细进度对比分析</h2>

# 【2.1 关键任务跟踪】
<h3>2.1 关键任务跟踪</h3>
<p>列出关键任务的完成情况，标注偏差，关注即将到期的任务。</p>

# 【2.2 里程碑达成情况】
<h3>2.2 里程碑达成情况</h3>
<p>列出里程碑的计划和实际日期，标注状态，关注下一个即将到来的里程碑。</p>

# 【第三章 问题与风险分析】— 滞后任务和风险
<h2>三、问题与风险分析</h2>

# 【3.1 滞后任务清单】
<h3>3.1 滞后任务清单</h3>
<p>列出滞后任务、滞后天数。如果有责任人/前置任务等数据则合理推断原因，否则如实标注"数据不足以判断原因"。</p>

# 【3.2 主要问题说明】
<h3>3.2 主要问题说明</h3>
<p>说明最严重的 2-3 个问题及其影响程度。</p>

# 【3.3 风险评估】
<h3>3.3 风险评估</h3>
<p>当前主要风险、风险等级、应对措施。</p>

# 【第四章 下一步行动计划】— 整改措施
<h2>四、下一步行动计划</h2>

# 【4.1 核心目标】
<h3>4.1 核心目标</h3>
<p>下周计划完成什么，达到什么进度。</p>

# 【4.2 重点工作安排】
<h3>4.2 重点工作安排</h3>
<p>具体任务、责任人、时间。</p>

# 【4.3 问题整改计划】
<h3>4.3 问题整改计划</h3>
<p>针对滞后任务的具体整改措施。</p>

## 语言要求
- 用中文
- 结论先行，先讲核心再讲细节
- 每条建议都要具体可执行
- 适当使用 🟢🟡🔴 标注状态

**只输出 HTML 内容，不要用 ```html 包裹。不要有 html/head/body 标签。**

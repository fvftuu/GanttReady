## 🔧 技术细节

### 项目结构
```
src/NetPlan.Server/
├── Models/              # 数据实体
├── Data/                # EF Core DbContext
├── Services/            # 业务逻辑层
├── Pages/               # Blazor 页面
│   ├── Index.razor      # 项目列表
│   └── Project/

│       ├── Network.razor # 网络图
│       ├── Resources.razor # 资源管理
│       └── Analysis.razor # 计划分析
└── wwwroot/js/         # 前端脚本
    └── cytoscape-dagre.js # 网络图布局
```

### 关键算法

**关键路径法（CPM）：**
1. 正向计算：从起始节点计算各节点的最早时间（ES, EF）
2. 逆向计算：从结束节点计算各节点的最晚时间（LS, LF）
3. 时差计算：总时差 = LS - ES = LF - EF
4. 关键路径：时差为 0 的节点组成的最长路径

### 部署方式

**单机部署：**
```powershell
cd src/NetPlan.Server
dotnet publish -c Release
# 复制输出目录到目标机器
dotnet NetPlan.Server.dll
```

**局域网部署：**
1. 在服务器上运行应用
2. 修改 `appsettings.json` 中的 `Urls` 为服务器 IP
3. 客户端浏览器访问 `http://服务器IP:5000`

**SQLite 共享方案：**
- 将 SQLite 数据库文件放在共享文件夹
- 所有用户连接到同一个数据库文件
- 注意：SQLite 不支持真正的并发写入，高并发场景建议迁移到 SQL Server

## 用户需求（严格按照四步执行）

### 第一步：计算所有时间参数
- 计算最早开始时间(ES)、最早完成时间(EF)
- 计算最迟开始时间(LS)、最迟完成时间(LF)
- 计算总时差(TF=LS-ES)和自由时差(FF=紧后ES-本工作EF)
- 识别关键线路：总时差为0的工作组成的线路

**状态**：✅ 已在 `ScheduleEngine.cs` 中完成计算，数据已传递到前端。

### 第二步：垂直分层布局（最重要）
1. 将所有工作按逻辑关系分成不同的垂直层级
2. 同一层级的工作y坐标相同，不同层级y坐标相差3行Excel单元格
3. 有紧前紧后关系的工作，紧后工作放在紧前工作的下一层
4. 平行工作放在同一层或不同层，绝对不允许重叠
5. 关键线路工作放在最上层，非关键工作依次向下排列

**实现方案**：
- 新的 `calculateVerticalLayout()` 函数
- 关键线路(TF=0) → 第0层（最上层）
- 非关键工作 → 按与关键线路的距离分层（距离+1）
- 每层内按ES排序，X坐标 = ES × 2 + 边距
- 检查同层节点X轴重叠，自动调整间距

**状态**：✅ 已完成

### 第三步：绘制规范的网络图
1. 时间坐标轴在最顶部，每个时间单位占2列Excel单元格
2. 节点用直径1.5cm的圆形，内部标注节点编号，节点中心严格对准时间刻度
3. 实箭线：水平段长度=持续时间×2列，线宽1.5磅，箭线上方标注工作代号和名称，下方标注持续时间
4. 虚箭线：垂直绘制，线宽1磅，虚线样式，不占时间
5. 波形线：黄色虚线，水平段长度=自由时差×2列，线宽1磅
6. 关键线路：红色粗线(3磅)，关键节点用红色填充
7. 字体统一使用宋体10号，文字居中对齐，绝对不允许重叠
8. 所有箭线和节点都可编辑

**实现方案**：
- 时间坐标轴：`renderTimeRuler()` 已在顶部绘制
- 节点：圆形，NODE_R=28（直径56px ≈ 1.5cm），内部显示 eventNum
- 实箭线：L形，水平段长度 = duration × dayWidth
- 虚箭线：垂直绘制，线型 `[6, 4]`
- 波形线：黄色(#FFD700)，使用自由时差(FF)，`waveLineTo()` 函数
- 关键线路：红色(#e63946)，线宽3磅，节点红色填充
- 字体：SimSun（宋体）10号
- 可编辑：双击节点打开编辑对话框

**状态**：✅ 已完成

### 第四步：添加图例和说明

## 关键设计规范

### 时标网络图规范（JGJ/T 121-2015）
1. **时间坐标**：横向，顶部，时间单位对齐节点中心
2. **节点**：圆形，直径1.5cm，内部编号
3. **实箭线**：水平段长度 = 持续时间 × 时间单位宽度
4. **虚箭线**：垂直，表示逻辑关系
5. **波形线**：黄色，表示自由时差
6. **关键线路**：红色粗线

### 布局算法
- 关键线路 → 最上层（第0层）
- 非关键工作 → 按与关键线路的距离向下分层
- 同层节点按ES排序，X坐标基于时标
- 避免重叠：检查同层节点间距 ≥ MIN_X_GAP(60px)

1. **布局算法优化**：
   - 当前分层算法基于"到关键路径的距离"
   - 可能需要根据实际的紧前紧后逻辑关系进一步优化

2. **文字重叠预防**：
   - 当前有基本检查（文本截断）
   - 可能需要更智能的文字位置调整算法

3. **跨层箭线优化**：
   - 当前使用L形箭线（先水平再垂直）
   - 可能需要根据具体情况选择更好的路径

## 技术要点

### 数据传递
- 后端计算：ES/EF/LS/LF/TF/FF
- 前端接收：JSON格式，包含 `es, ef, ls, lf, tf, ff` 字段
- 自由时差：使用 `ff` 字段绘制波形线

### 坐标计算
- X坐标（时标）：`x = es * dayWidth + MARGIN_X`
- Y坐标（分层）：`y = 80 + layer * RANK_HEIGHT`
- 节点中心对准时间刻度

### 绘图顺序
1. 时间标尺（最底层）
2. Cytoscape节点（圆形）
3. Canvas覆盖层：
   - 箭线（实线、虚线）
   - 波形线（黄色）
   - 文字标注
   - 图例
   - 总工期说明

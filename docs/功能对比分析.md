# 进度计划项目 - 技术方案比对分析

## 项目需求概览

| 功能模块 | 需求描述 |
|---------|---------|
| 1. 进度可视化 | 甘特图计划 + 时标双代号网络计划 |
| 2. 任务管理 | 可编辑：项目代号、任务名称、计划开始/完成/工期、实际开始/完成/工期 |
| 3. 资源管理 | 可存取的人、材、机数据库 |
| 4. 逻辑关系 | 可编辑时差的任务时间关系：SS、SF、FS、FF |
| 5. 分析功能 | 工期总体指标、资源使用汇总/明细、工序进度明细 |
| 6. 界面灵活 | 可增减的列 |

## 技术方案对比表

### 方案A：WPF + DevExpress（推荐用于桌面专业软件）

| 评估维度 | 详细说明 | 评分 |
|---------|---------|------|
| **技术栈** | WPF + C# + .NET 6/8 + DevExpress控件库 | ⭐⭐⭐⭐ |
| **开发难度** | 中等（您已熟悉WPF），DevExpress提供成熟甘特图控件 | ⭐⭐⭐⭐ |
| **甘特图实现** | DevExpress GanttControl（商业控件，功能完整）或Syncfusion Gantt | ⭐⭐⭐⭐⭐ |
| **网络图实现** | 需自建或使用GraphSharp等图形库，较复杂 | ⭐⭐⭐ |
| **数据存储** | SQLite（本地）+ JSON配置（列定义） | ⭐⭐⭐⭐ |
| **动态列** | DevExpress TableView支持运行时列增删 | ⭐⭐⭐⭐⭐ |
| **资源数据库** | Entity Framework + SQLite，支持CRUD | ⭐⭐⭐⭐ |
| **逻辑关系** | 自建Task类，支持SS/SF/FS/FF + Lag(时差) | ⭐⭐⭐⭐ |
| **分析功能** | C#直接计算，支持导出Excel | ⭐⭐⭐⭐ |
| **部署方式** | 单机EXE/MSI安装包 | ⭐⭐⭐⭐ |
| **成本** | DevExpress商业授权（约¥8,000-15,000/年）或免费替代方案 | ⭐⭐ |
| **性能** | 原生桌面，万级任务无压力 | ⭐⭐⭐⭐⭐ |
| **维护性** | 强类型，重构友好，您已熟悉 | ⭐⭐⭐⭐⭐ |

**优势**：
- ✅ 您已有WPF经验，上手快
- ✅ 桌面软件，离线可用，适合工地现场
- ✅ DevExpress甘特图控件专业，支持编辑、拖拽
- ✅ 动态列、排序、筛选原生支持

**劣势**：
- ❌ 商业控件成本高（可用免费替代：OxyPlot+自建，但工作量大）
- ❌ 仅Windows平台
- ❌ 网络图需额外开发

---

### 方案B：WPF + 开源控件（低成本方案）

| 评估维度 | 详细说明 | 评分 |
|---------|---------|------|
| **技术栈** | WPF + C# + .NET 6/8 + OxyPlot/MaterialDesign | ⭐⭐⭐⭐ |
| **开发难度** | 较高（需自建甘特图） | ⭐⭐⭐ |
| **甘特图实现** | 基于OxyPlot或WPF Canvas自建，工作量较大 | ⭐⭐⭐ |
| **网络图实现** | 使用QuickGraph+GraphSharp或AvalonEdit | ⭐⭐⭐ |
| **数据存储** | SQLite + Dapper（轻量级ORM） | ⭐⭐⭐⭐ |
| **动态列** | DataGrid动态列（较麻烦，需反射） | ⭐⭐⭐ |
| **资源数据库** | SQLite + JSON序列化物料清单 | ⭐⭐⭐⭐ |
| **逻辑关系** | 自建调度引擎（关键路径算法） | ⭐⭐⭐ |
| **分析功能** | 需自建计算和报表生成 | ⭐⭐⭐ |
| **部署方式** | 单机EXE | ⭐⭐⭐⭐ |
| **成本** | 免费（全部开源） | ⭐⭐⭐⭐⭐ |
| **性能** | 自建控件需优化，千级任务OK | ⭐⭐⭐ |
| **维护性** | 代码量大，维护成本较高 | ⭐⭐⭐ |

**优势**：
- ✅ 零成本
- ✅ 完全可控，无黑盒

**劣势**：
- ❌ 开发周期长（甘特图自建需2-3个月）
- ❌ 用户体验不如商业控件
- ❌ 需自建关键路径算法

---

### 方案C：Web应用（React/Vue + .NET WebAPI）

| 评估维度 | 详细说明 | 评分 |
|---------|---------|------|
| **技术栈** | React/Vue + TypeScript + .NET 8 WebAPI + PostgreSQL/SQLite | ⭐⭐⭐ |
| **开发难度** | 中等偏高（前后端分离） | ⭐⭐⭐ |
| **甘特图实现** | DHTMLX Gantt（商业JS库）或ECharts自建 | ⭐⭐⭐⭐ |
| **网络图实现** | vis.js / Cytoscape.js（专业图形库） | ⭐⭐⭐⭐⭐ |
| **数据存储** | 后端数据库（PostgreSQL/MySQL） | ⭐⭐⭐⭐ |
| **动态列** | 前端动态渲染，天然支持 | ⭐⭐⭐⭐⭐ |
| **资源数据库** | RESTful API + 前端SPA | ⭐⭐⭐⭐ |
| **逻辑关系** | 后端C#计算引擎 + 前端可视化 | ⭐⭐⭐⭐ |
| **分析功能** | 后端计算 + 前端Chart.js展示 | ⭐⭐⭐⭐ |
| **部署方式** | Web服务器（IIS/Nginx）或云服务 | ⭐⭐⭐ |
| **成本** | DHTMLX商业授权（约$790）或免费替代 | ⭐⭐⭐ |
| **性能** | 依赖网络，大量数据需分页/虚拟滚动 | ⭐⭐⭐ |
| **维护性** | 前后端分离，团队协作友好 | ⭐⭐⭐⭐ |

**优势**：
- ✅ 跨平台（Windows/Linux/macOS）
- ✅ 网络图有成熟JS库（Cytoscape）
- ✅ 动态列天然支持
- ✅ 可多用户协作

**劣势**：
- ❌ 需要服务器部署
- ❌ 离线使用需PWA或Electron封装
- ❌ 您可能需要学习前端技术栈

---

### 方案D：Blazor + .NET全栈（现代WebAssembly方案）

| 评估维度 | 详细说明 | 评分 |
|---------|---------|------|
| **技术栈** | Blazor WebAssembly + .NET 8 + SQLite（本地） | ⭐⭐⭐⭐ |
| **开发难度** | 中等（全C#，无JS） | ⭐⭐⭐⭐ |
| **甘特图实现** | 使用Blazor版Gantt组件或JS互操作 | ⭐⭐⭐⭐ |
| **网络图实现** | Blazor+Cytoscape.js互操作 | ⭐⭐⭐⭐ |
| **数据存储** | SQLite（本地）或IndexedDB（浏览器） | ⭐⭐⭐⭐ |
| **动态列** | Razor组件动态渲染 | ⭐⭐⭐⭐ |
| **资源数据库** | EF Core + SQLite | ⭐⭐⭐⭐ |
| **逻辑关系** | C#计算引擎（共享WPF逻辑） | ⭐⭐⭐⭐⭐ |
| **分析功能** | C#后端计算 + Blazor界面展示 | ⭐⭐⭐⭐ |
| **部署方式** | 静态文件部署（GitHub Pages/Azure Static Web Apps） | ⭐⭐⭐⭐⭐ |
| **成本** | 免费（.NET开源） | ⭐⭐⭐⭐⭐ |
| **性能** | WebAssembly首次加载慢，后续流畅 | ⭐⭐⭐ |
| **维护性** | 全C#，您已熟悉，维护成本低 | ⭐⭐⭐⭐⭐ |

**优势**：
- ✅ 全C#开发，无JS学习成本
- ✅ 可部署为桌面（Blazor Hybrid + MAUI）
- ✅ 代码可复用（WPF和Blazor共享业务逻辑）
- ✅ 跨平台

**劣势**：
- ❌ WebAssembly首次加载慢（需优化）
- ❌ 甘特图组件生态不如JS丰富
- ❌ Blazor相对较新，社区资源少

## 推荐方案

### 🥇 第一推荐：方案A（WPF + DevExpress）

**理由**：
1. 您已有WPF经验，开发效率高
2. DevExpress Gantt控件专业，支持所有需求
3. 桌面软件适合工地现场（可离线）
4. 性能最优，万级任务无压力

**成本考虑**：
- 如果预算有限，可用 **Syncfusion Community License**（年收入<100万美元可免费）
- 或采用 **方案B**，用OxyPlot自建甘特图（工作量增加2-3个月）

---

### 🥈 第二推荐：方案D（Blazor + .NET全栈）

**理由**：
1. 全C#开发，无学习成本
2. 可部署为Web或桌面（MAUI）
3. 跨平台，未来扩展性强
4. 成本最低（全部开源）

**适合场景**：
- 如果需要多用户协作
- 如果未来需要云端部署
- 如果想要现代化UI

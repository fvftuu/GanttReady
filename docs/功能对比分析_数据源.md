# 进度计划项目 - 开源技术方案比对（更新版）

## 约束条件
- ❌ 不接受商业控件
- ✅ 部署目标：单机 + 局域网
- ✅ 技术栈偏好：.NET生态

---

## 方案对比（仅开源方案）

### 方案1：WPF + 开源控件库（成本优先）

| 评估维度 | 详细说明 | 评分 |
|---------|---------|------|
| **技术栈** | WPF + C# + .NET 8 + 开源控件库 | ⭐⭐⭐⭐ |
| **甘特图实现** | **OxyPlot**（绘图）+ **WPF Toolkit DataGrid**（表格）<br/>或 **Live-Charts2**（备选） | ⭐⭐⭐ |
| **网络图实现** | **GraphSharp**（WPF图形库）或 **AvalonEdit**（文本图形） | ⭐⭐⭐ |
| **数据存储** | **SQLite** + **Dapper**（轻量级ORM） | ⭐⭐⭐⭐⭐ |
| **动态列** | DataGrid动态列（需代码 Behind 支持） | ⭐⭐⭐ |
| **资源数据库** | SQLite + JSON序列化扩展字段 | ⭐⭐⭐⭐ |
| **逻辑关系** | 自建调度引擎（FS/SS/SF/FF + Lag） | ⭐⭐⭐⭐ |
| **分析功能** | C#计算 + **EPPlus**（开源Excel导出） | ⭐⭐⭐⭐ |
| **部署方式** | 单机EXE + 局域网文件共享（SQLite支持） | ⭐⭐⭐⭐ |
| **成本** | **零成本**（全部开源） | ⭐⭐⭐⭐⭐ |
| **性能** | 原生WPF，优化后千级任务流畅 | ⭐⭐⭐⭐ |
| **维护性** | 您已熟悉WPF，维护成本低 | ⭐⭐⭐⭐⭐ |
| **开发周期** | **3-5个月**（甘特图需自建） | ⭐⭐ |

**开源组件推荐**：
- **OxyPlot**：绘图库，可绘制甘特图时间轴
- **WPF Toolkit**：DataGrid支持排序/筛选/分组
- **GraphSharp**：图形布局引擎（有向图）
- **Dapper**：轻量级ORM，性能优于EF
- **EPPlus**：Excel导出（开源替代MS Interop）

**优势**：
- ✅ 零成本
- ✅ 您已有WPF经验
- ✅ 单机部署简单
- ✅ 完全可控

**劣势**：
- ❌ 甘特图需自建（工作量最大）
- ❌ 网络图库较老（GraphSharp最后更新2015年）
- ❌ 动态列实现较麻烦

### 方案2：Blazor Server（跨平台 + 局域网部署）

| 评估维度 | 详细说明 | 评分 |
|---------|---------|------|
| **技术栈** | Blazor Server + .NET 8 + Razor组件 | ⭐⭐⭐⭐⭐ |
| **甘特图实现** | **Blazor-Gantt**（开源组件）或 **JS互操作** + DHTMLX Community | ⭐⭐⭐⭐ |
| **网络图实现** | **Cytoscape.js**（通过JS互操作） | ⭐⭐⭐⭐⭐ |
| **数据存储** | **SQLite**（本地）或 **PostgreSQL**（局域网） | ⭐⭐⭐⭐⭐ |
| **动态列** | Razor组件动态渲染（天然支持） | ⭐⭐⭐⭐⭐ |
| **资源数据库** | EF Core + SQLite | ⭐⭐⭐⭐⭐ |
| **逻辑关系** | C#服务端计算引擎 | ⭐⭐⭐⭐⭐ |
| **分析功能** | 服务端计算 + **Chart.js** 可视化 | ⭐⭐⭐⭐⭐ |
| **部署方式** | **局域网IIS/Nginx** 或 **单机自宿主（Kestrel）** | ⭐⭐⭐⭐⭐ |
| **成本** | **零成本**（全部开源） | ⭐⭐⭐⭐⭐ |
| **性能** | 局域网内延迟<50ms，用户体验接近桌面 | ⭐⭐⭐⭐ |
| **维护性** | 全C#，您已熟悉，代码复用率高 | ⭐⭐⭐⭐⭐ |
| **开发周期** | **2-3个月**（有现成Blazor甘特图组件） | ⭐⭐⭐⭐ |

**开源组件推荐**：
- **Blazor Gantt Chart**（GitHub开源）：https://github.com/BlazorComponents/Blazor-Gantt
- **Cytoscape.js**：专业网络图库（支持有向图、力导向布局）
- **Chart.js**：图表可视化
- **EF Core + SQLite**：轻量级ORM

**部署方式（局域网）**：
```
方案A：IIS部署（Windows Server）
  用户通过浏览器访问 http://server-ip:5000

方案B：自宿主（Kestrel）
  双击运行 NetPlan.Server.exe
  局域网其他电脑访问 http://主机IP:5000

方案C：Docker容器（跨平台）
  docker run -p 5000:80 netplan:latest
```

**优势**：
- ✅ 零成本
- ✅ 全C#开发（无JS学习成本）
- ✅ 动态列天然支持
- ✅ 网络图有成熟JS库（Cytoscape）
- ✅ 可扩展为多用户（未来需求）
- ✅ 部署灵活（IIS/自宿主/Docker）

**劣势**：
- ❌ 需要浏览器（但可PWA离线）
- ❌ 首次加载需下载WebAssembly（约5-10MB）

### 方案3：Blazor Hybrid（MAUI）+ WPF风格（桌面原生）

| 评估维度 | 详细说明 | 评分 |
|---------|---------|------|
| **技术栈** | Blazor Hybrid（MAUI） + .NET 8 + Razor | ⭐⭐⭐⭐ |
| **甘特图实现** | 同方案2（Blazor组件复用） | ⭐⭐⭐⭐ |
| **网络图实现** | 同方案2（Cytoscape.js） | ⭐⭐⭐⭐⭐ |
| **数据存储** | SQLite（本地文件） | ⭐⭐⭐⭐⭐ |
| **动态列** | Razor组件动态渲染 | ⭐⭐⭐⭐⭐ |
| **资源数据库** | EF Core + SQLite | ⭐⭐⭐⭐⭐ |
| **逻辑关系** | 共享方案2的C#计算引擎 | ⭐⭐⭐⭐⭐ |
| **分析功能** | 同方案2 | ⭐⭐⭐⭐⭐ |
| **部署方式** | **MSI安装包**（Windows）或 **APPX**（Microsoft Store） | ⭐⭐⭐⭐ |
| **成本** | **零成本** | ⭐⭐⭐⭐⭐ |
| **性能** | 原生桌面性能（WebView2嵌入） | ⭐⭐⭐⭐⭐ |
| **维护性** | 一套代码，多平台部署（Windows/macOS/Android） | ⭐⭐⭐⭐⭐ |
| **开发周期** | **2-3个月**（与方案2共享组件） | ⭐⭐⭐⭐ |

**说明**：
- Blazor Hybrid = WPF的现代化替代
- 使用WebView2渲染（Chromium内核）
- UI用Razor组件（类似React/Vue）
- 业务逻辑用C#（与WPF共享）

**优势**：
- ✅ 零成本
- ✅ 原生桌面体验
- ✅ 一套代码，多平台部署
- ✅ 未来可扩展到移动端

**劣势**：
- ❌ MAUI相对较新（有坑）
- ❌ WebView2需安装（约300MB）

## 推荐方案

### 🥇 第一推荐：方案2（Blazor Server）

**理由**：
1. **零成本**：全部开源
2. **开发快**：有现成的Blazor甘特图组件
3. **网络图强**：Cytoscape.js专业图形库
4. **动态列天然支持**：Razor组件灵活
5. **部署灵活**：局域网IIS / 单机自宿主均可
6. **未来扩展**：可轻松加多用户协作

**部署架构（局域网）**：
```
[用户电脑] ──HTTP──> [局域网服务器]
                        └─ NetPlan (Blazor Server)
                             └─ SQLite数据库

或（单机模式）：

[用户电脑] ──localhost──> [Kestrel自宿主]
                                └─ NetPlan.exe
                                     └─ SQLite数据库
```

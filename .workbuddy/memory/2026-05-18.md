## 修复阶段完成率：OutlineLevel→Code层级判定 (Task #59)
- **问题**: 恒力项目 54 任务分布 Lv1~4，但 OutlineLevel 数据不一致（如 1.2.1 Lv=2 但 1.2.2 Lv=1），按 OutlineLevel 分组产生 4 个无意义阶段
- **根因**: OutlineLevel 依赖手动录入，不可靠。真正的阶段结构在 WBS 代码中（1.1施工准备 / 1.2梁板预制 / 1.3泊位 / 1.4通用泊位 / 1.5泵房）
- **修复**:
  1. 新增 `HasCodeHierarchy`：只有代码深度不一（如 1 / 1.2 / 1.3.1.1）才视为层级结构
  2. 新增 `DetermineCodeGroupingLevel`：在 code 层级中找到 ≥2 组各有 ≥2 任务的有效分组层级
  3. 新增 `GroupByCodePrefix` / `GroupByCodePrefixAndMilestone`：按代码前缀分组替代 OutlineLevel
  4. 新增 `SplitCodeParts`：兼容 "1.3.1.1" 数字点分和 "A1"/"B2" 字母数字两种格式
  5. 过滤单任务组（父级汇总任务如 "1"）避免产生无意义阶段
- **验证结果**:
  - 项目1（综合基础设施，A1-E6 全 Lv1）：hasCodeHierarchy=False → "按里程碑" → 7 阶段
  - 项目3（恒力，1~1.5.x.x）：hasCodeHierarchy=True → "按代码层级" → 4 阶段（1.2/1.3/1.4/1.5）
- **构建**: 0错误 0警告，服务 `http://localhost:5000`

## 手动排程保护 + 状态栏实时更新

### 需求
1. 手动调整进度条后点击"重算计划"，进度条和日期不应变回初始状态
2. 状态栏的总工期需随进度条调整实时变化

### 根因
- 问题1: CPM 引擎 `CalculateEarlyTimes` 总是按依赖关系重算，忽略手动拖拽的日期
- 问题2: 状态栏总工期用项目 `PlanEndDate`，拖拽后未刷新

### 修复

**Models/TaskItem.cs**: 新增 `IsManualSchedule` 字段 (bool, 默认 false)

**Services/ScheduleEngine.cs**: `CalculateEarlyTimes` 中对手动排程任务：
- 计算 `manualEarlyStart = (PlanStartDate - projectStartDate).Days`
- `EarlyStart = max(CPM计算值, manualEarlyStart)`
- 即紧前任务可推后它，但不会早于手动设置的日期

**Services/ProjectService.cs**: `UpdateTaskAsync` 新增 `existing.IsManualSchedule = task.IsManualSchedule`

**Pages/Project/Gantt.razor**:
- `OnBarDragEnd`: 拖拽保存时设置 `IsManualSchedule = true`
- 状态栏: 总工期改为 `tasks.Max(t => t.PlanEndDate) - projectStartDate` 实时计算
- 编辑对话框: 新增 🔒手动排程 checkbox（可手动清除标记）

### 构建

## 修复网络图纵向滚动（根因：.page-main 不是 flex 容器）

### 问题
`#network-body` 设置了 `overflow-y: auto` 但不滚动，因为父级 `.page-main` 是 `display: block`，
导致 `.network-wrapper` 的 `flex: 1` 不生效，整条链撑破到 content-height 而非 viewport-height。

### 根因定位（playwright-cli 浏览器实测）
```
修前: .page-main:       offset=672 scroll=2404 ← 内容撑破了!
      .network-wrapper:  offset=2356              ← flex:1 在 block 上下文中无意义
      #network-body:     offset=2342 scroll=2342   ← 无滚动!
修后: .page-main:       offset=672 scroll=672   ← 约束成功
      .network-wrapper:  offset=624              ← flex:1 生效
      #network-body:     offset=610 scroll=2342   ← 可滚动!
```

### 修复：`site.css` .page-main
```css
.page-main {
    flex: 1;
    display: flex;          /* 新增：作为 flex 容器 */
    flex-direction: column; /* 新增：纵向排列 */
    overflow: auto;
    min-height: 0;          /* 新增：允许 flex 子项收缩 */
}
```

### 验证
- playwright-cli 确认 SVG 渲染正常（16340×2342px）
- #network-body scrollHeight=2342 > offsetHeight=610，可滚动
- 今日线、前锋线 DOM 元素均存在

## 甘特图拖拽防上移

### 问题
拖拽甘特图进度条时条形自动上移（未保持水平移动）

### 根因
`JS.InvokeVoidAsync("hideDragGhost", e)` 中 `DragEventArgs` 被 Blazor 序列化为 JSON 传到 JS，
`e.dataTransfer` 不是原生浏览器 DataTransfer 对象，`setDragImage()` 未生效。
浏览器使用默认鬼影（截图跟随鼠标），导致视觉上条形跳动。

### 修复
- **wwwroot/js/netplan.js**: 替换为原生 `document.addEventListener('dragstart', ...)`，用 `e.target.closest('.gantt-bar-frame')` 找到条形元素，计算 `offsetX`/`offsetY`（鼠标相对于条形的偏移），将 1px 透明鬼影定位到条形同一行
- **Pages/Project/Gantt.razor**: 移除 `_ = JS.InvokeVoidAsync("hideDragGhost", e)` 无效调用

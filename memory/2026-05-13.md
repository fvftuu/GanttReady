# 2026-05-13 Memory

## Bug Fixes & Features Implemented

### Bug 1 — ScheduleEngine.cs: Manual schedule vs predecessor conflict
- When a task has **both** a manually-set start date AND predecessor relationships, the old code took `Max(CPM_ES, manual_ES)`.
- **Fix**: If predecessors exist → use pure CPM; only apply manual start when there are **no** predecessors.
- File: `ScheduleEngine.cs` lines 121-124

### Bug 2 — Gantt chart time scale offset
- `predEndOffset` at line 348 lacked `Math.Max(0, ...)` protection. If predecessor ends before project start, SVG arrow enters negative X → browser expands viewport leftward, showing dates before project start.
- **Fix**: Added `Math.Max(0, ...)` to `predEndOffset`; added explicit width to SVG element to prevent left expansion.

### Feature: Predecessor/Successor lag display
- Gantt left table and edit dialog now show `A01+5` / `A01-3` format (code + lag).
- Lag=0 shows plain code.
- Files: `GanttTable.razor`, `Gantt.razor`

### Feature: Gantt time scale start date
- **Reversal**: User wants earliest date among all task PlanStartDate/ActualStartDate.
- Changed from always-project-start to: `Min(project.PlanStartDate, all task PlanStartDate, all task ActualStartDate)`
- End date similarly extends: `Max(project.PlanEndDate, all task PlanEndDate, all task ActualEndDate)`
- File: `Gantt.razor` lines 1060-1082

### Feature: Dummy arrow node connection
- Dummy arrows now start from source node RIGHT EDGE (`src.x + NODE_R`) instead of center.
- Updated both `buildNetworkSvg` and `_netUpdateDummys` drag handler.
- File: `netplan.js`

### Feature: Task name indent on outline level
- Task name column now indents like code column: `padding-left: Math.Max(0,(OutlineLevel-1)*16)px`.
- Added `word-break: break-all` for auto-wrap.
- Row height: `height: 36px` → `min-height: 36px` with dynamic row expansion.
- `rowHeight` in Gantt: 36px → 48px.
- File: `GanttTable.razor`, `site.css`, `Gantt.razor`

### Feature: Legend position
- Moved from bottom-right (`cw-220, ch-75`) to bottom-left (`10, bottomRulerY-85`).
- Legend completely above bottom time ruler.
- File: `netplan.js`

### Feature: Arrow anti-overlap
- Changed from "same source+target" grouping to "same source node" grouping.
- Offset: `±NODE_R*0.8` → `±30px` (half layer height).
- File: `netplan.js`

### Feature: Arrow cross-over detection (bridge arcs)
- Added `findSegIntersection` using quick-reject + cross product tests.
- Post-processes all activity arrow line segments for intersections.
- Non-critical arrow intersections get a bridge arc (semicircle).
- Critical-line intersections are skipped (no bridge on critical path).
- File: `netplan.js`

### Feature: Gantt bar height adjustment
- Plan bar (50%) + act bar (50%) = 100% of row height.
- `barHeight = rowHeight * 0.9` → `rowHeight` (48px).
- File: `Gantt.razor`

## Key Decisions
- User prefers explicit precision: "审题仔细", no assumptions about intent.
- Dummy arrows in dual-code diagrams: all successors are REAL WORK. Dummy arrow is the logical connector spanning time gaps (e.g., FS+5).
- Event merging must preserve logical path separation — even if same time offset, don't merge events from different paths.
- Legend at bottom-left, not bottom-right or top-left.
- Arrow overlap fix by full or half-layer offset, not small pixel offsets.

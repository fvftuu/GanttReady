**Bridge Architecture (openspace-mcp-bridge.js):**
- Node.js HTTP server wrapping openspace-mcp stdio as HTTP callback.
- Port 4876, auth via secret `openspace-bridge-secret` (header: `x-mcp-bridge-secret`).
- MCP protocol flow: initialize → notifications/initialized → tools/list → tools/call {name: toolName, arguments: {...}}.
- **Critical: method must be "tools/call" with params {name, arguments}, NOT `method: toolName`.**

**Bridge Debugging Saga:**
- Initial 500 errors: Node.js `spawn` + pipe stdio, `mcpProcess.connected` is false for piped processes, used `mcpProcess !== null && mcpProcess.stdin.writable`.
- Bridge initializes MCP on boot (send initialize after 3s delay). Auto-restarts on crash.
- Tested and verified: `search_skills` returns `{"results":[],"count":0}`, `execute_task` returns successfully.

## Key Decisions

- Bridge script handles MCP init sequence + auto-restart. Auth via pre-shared secret.
- DEEPSEEK_API_KEY passed via environment for OpenSpace's model access.

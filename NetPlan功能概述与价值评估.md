## 一、项目概述

### 1.1 项目定位

NetPlan 是一款**开源的网络计划编制与进度管理软件**，专注于**中国国标 GB/T 13400** 的双代号网络图（AOA）实现，填补了开源项目管理工具在**国标AOA网络图**领域的空白。

### 1.2 目标用户

| 用户类型 | 规模 | 核心需求 | NetPlan价值主张 |
|---------|------|---------|----------------|
| 建筑/工程企业 | 中型 | 国标AOA网络图，成本敏感 | 免费开源，国标合规 |
| 小型施工队 | 小型 | 简单易用的项目管理 | 开源免费，上手快 |
| 大型国企/政府 | 大型 | 私有化部署，数据安全 | 私有化+定制开发 |
| 高校/培训机构 | 教育 | 教学需要，预算有限 | 开源免费，教学友好 |
| 个人用户 | 个人 | 学习项目管理 | 免费+功能完整 |

### 1.3 核心价值主张

1. **填补市场空白**：目前**没有开源项目**完整实现 GB/T 13400 标准的双代号网络图（AOA）
2. **现代化Web实现**：传统软件（维达/梦龙/同望）都是桌面软件且界面陈旧，NetPlan 用 Blazor Server 实现现代化Web应用
3. **成本优势**：相比 P6（数万元/年）和广联达（昂贵），NetPlan 开源免费
4. **技术栈优势**：.NET 8 + Blazor Server，企业级，易部署，易扩展

### 2.3 核心算法

#### CPM（关键路径法）引擎
- **正向传播**：计算 ES（最早开始）、EF（最早完成）
- **反向传播**：计算 LS（最迟开始）、LF（最迟完成）
- **时差计算**：TF（总时差）= LS - ES，FF（自由时差）= 后续ES - EF
- **关键路径识别**：TF = 0 的任务组成关键路径
- **手动排程保护**：IsManualSchedule 字段，CPM 计算时取 max(CPM, manual)

#### 网络图布局算法
- **事件合并**：同一时间偏移的任务 ES/EF 合并为同一事件节点
- **BFS分层**：关键路径节点固定 layer=1，其余节点按前驱迭代推进
- **时标坐标系**：X = MARGIN_LEFT + timeOffset * dayWidth；Y = 52 + 16 + (layer-1) * 60

### 3.1 甘特图模块（Gantt.razor）

#### 已完成功能 ✅
1. **冻结窗格**：左侧任务列表固定，右侧甘特图可横向滚动
2. **时间标尺**：双层结构，根据 dayWidth 自动切换日/周/月模式
3. **周末高亮**：周六日灰底(#f0f0f0) + 灰字(#999)
4. **任务交互**：
   - 双击任务行打开编辑对话框
   - 拖拽调整任务日期
   - 任务选中高亮
5. **条形图颜色规则**（5种状态）：
   - 未开始(蓝)、进行中(橙)、已完成(绿)、关键(红)、里程碑(紫)

6. **缩放控制**：+ / - 按钮，每次5%幅度，范围5%-1500%
7. **适应页面**：自动计算最佳缩放比例

### 4.2 主要竞品分析

#### 4.2.1 传统国产软件（维达/梦龙/同望）
- **优势**：深耕行业多年，符合国标，用户习惯成熟
- **劣势**：
  - 闭源商业，授权费用高
  - 界面陈旧（WinForm/MFC风格）
  - 不支持Web/移动端
  - 数据孤岛，难与其他系统集成
- **NetPlan的机会**：用现代Web技术 + 开源策略吸引用户

#### 4.2.2 广联达斑马进度计划
- **优势**：有"一表多图"同步技术，前锋线功能强
- **劣势**：闭源商业，价格高，生态封闭
- **NetPlan的机会**：开源 + 可扩展（SDK/API）

#### 4.2.3 国际商业软件（P6 / MS Project）
- **优势**：功能全面，生态成熟
- **劣势**：
  - **不支持中国国标AOA网络图**（致命弱点）
  - 贵（P6企业版数万元/年）
  - 学习曲线陡峭
- **NetPlan的机会**：专注中国市场需求，免费开源

#### 4.2.4 开源替代品（ProjectLibre / GanttProject）
- **优势**：免费开源
- **劣势**：
  - **不支持AOA双代号网络图**（NetPlan的核心差异化）
  - 功能相对基础
  - 界面不够现代化
- **NetPlan的机会**：用Blazor技术栈 + AOA国标实现差异化

# NetPlan 同步工具使用说明

## 这是什么

一个中文界面的同步工具，用 Git Bundle 方式在 U 盘上备份和恢复项目，
支持任意工作目录，完全离线可用。

## 如何运行

双击打开：

```
tools/
├── 启动同步工具.bat     ← 双击这个，弹出中文菜单
└── NetPlan-Sync.ps1    ← 核心脚本
```

## 使用流程

### 电脑 A：把代码备份到 U 盘

```
1. 打开项目目录 d:\99AI应用\NetPlan
2. 插入 U 盘
3. 双击 tools\启动同步工具.bat
4. 选择 U 盘盘符
5. 输入 1（备份到U盘）
   → 自动初始化 Git + 打包 Bundle
6. 把生成的 NetPlan.bundle 从 U 盘复制出来带走
```

### 电脑 B：从 U 盘恢复项目

```
1. 把 NetPlan.bundle 放到 U 盘根目录
2. 打开任意目录（路径随意，不一定要和电脑 A 一样）
3. 双击 tools\启动同步工具.bat
4. 选择 U 盘盘符
5. 输入 2（从U盘恢复）
6. 指定目标目录（直接回车默认桌面）
   → 项目克隆到本地
7. cd 到项目目录，运行：
   dotnet build
   dotnet run
```

### 以后同步（电脑 A 继续开发后）

```
电脑 A：选 1 备份到U盘
电脑 B：选 3 从U盘增量更新（自动合并最新代码）
```

## 菜单说明

| 选项 | 功能 | 场景 |
|------|------|------|
| 1 | 备份到U盘（Push） | 开发完成后同步 |
| 2 | 从U盘恢复（Pull） | 第一次在新电脑恢复 |
| 3 | 从U盘增量更新 | 已恢复过的项目拉取最新 |
| 4 | 查看同步状态 | 查看上次备份时间、提交数等 |

## 自动模式（无交互）

适合定期备份，不会弹出任何提示：

```powershell
.\NetPlan-Sync.ps1 -Auto
```

## 注意事项

- Bundle 文件包含**所有 Git 提交历史和分支**，无需 GitHub
- 同步时自动检测并提交未保存的更改
- 两台电脑工作目录可以不同，完全兼容
- Win10/11 自带 PowerShell，无需安装任何软件
